3) Como gerar “cores que combinam” a partir da seed
Sem complicar com lib externa, o caminho bom é trabalhar em HSL:
Primary = seed
Hover/Active = mesma matiz (H), aumenta/diminui L (lightness) e um pouco a
saturação
Bg/surface = neutros puxados levemente pra matiz do seed (fica “harmonioso”)
Border/ring = versões mais claras/dessaturadas do seed
Warning/Error/Info/Success:
você pode fixar matizes “universais” (warning≈35°, error≈5°, info≈205°,
success≈145°)
e ajustar saturação/lightness conforme mode
Isso gera um tema consistente mesmo com seed diferente.

function rgbToHsl(r: number, g: number, b: number) {
  r /= 255; g /= 255; b /= 255;
const max = Math.max(r, g, b), min = Math.min(r, g, b);
let h = 0, s = 0;
const l = (max + min) / 2;
const d = max - min;
if (d !== 0) {
    s = d / (1 - Math.abs(2 * l - 1));
switch (max) {
case r: h = ((g - b) / d) % 6; break;
case g: h = (b - r) / d + 2; break;
case b: h = (r - g) / d + 4; break;
    }
    h *= 60;
if (h < 0) h += 360;
  }
return { h, s: s * 100, l: l * 100 };
}
function hslToRgb(h: number, s: number, l: number) {
  s /= 100; l /= 100;
const c = (1 - Math.abs(2 * l - 1)) * s;
const x = c * (1 - Math.abs(((h / 60) % 2) - 1));
const m = l - c / 2;
let rp = 0, gp = 0, bp = 0;
if (0 <= h && h < 60) { rp = c; gp = x; bp = 0; }
else if (60 <= h && h < 120) { rp = x; gp = c; bp = 0; }
else if (120 <= h && h < 180) { rp = 0; gp = c; bp = x; }
else if (180 <= h && h < 240) { rp = 0; gp = x; bp = c; }
else if (240 <= h && h < 300) { rp = x; gp = 0; bp = c; }
else { rp = c; gp = 0; bp = x; }
return {
r: (rp + m) * 255,
g: (gp + m) * 255,
b: (bp + m) * 255,
  };
}

Se você curtir, eu faço o próximo upgrade: gerar também “tokens de componente” (ex.:--sg-btn-primary-bg , --sg-input-border , --sg-card-bg ) em cima dessas paletas. Aí
